Transcrição

Agora que preparamos o nosso ambiente, vamos iniciar a construção dos nossos códigos. Vamos começar resgatando o protótipo que fizemos no Figma.

Olhando para a nossa tela, temos o protótipo feito no Figma, e percebemos que ele tem diversas estilizações, além dos textos e imagens. Temos cores e também uma certa organização do texto. Isso significa que, além de um código HTML que estrutura a nossa página, temos também um código CSS que estiliza essa página. No entanto, ao longo desta aula, faremos somente o código HTML, ou seja, a estrutura base da nossa página.

Vamos dar um spoiler de como vai ficar a nossa página web ao final desta aula. Na nossa tela, temos nossa página web somente com HTML puro. Vejam a diferença. Aqui não temos cor e o texto também não está organizado. Os elementos não estão organizados na página como queremos no final da nossa construção.

Trabalhando com o VS Code

Mas por onde começamos? Precisamos primeiro ir até o nosso ambiente de desenvolvimento, o VS Code. E a nossa pasta já está aberta aqui, a pasta criada pelo Silvio no início.

Aqui dentro, vamos criar os nossos primeiros arquivos, nesse caso, o arquivo HTML. Temos algumas informações importantes sobre o nome desse arquivo. Primeiro, vamos criá-lo. Existem algumas formas de fazer isso, vamos mostrar uma delas.

Vamos no menu Explorador, no canto esquerdo da tela, primeiro símbolo, representado pelo ícone de duas folhas de papel sobrepostas. Temos algumas opções, conseguimos criar um Novo Arquivo, selecionado o ícone de uma folha de papel com um + na lateral inferior.

No símbolo ao lado, temos a opção de criar Nova Pasta. No canto esquerdo da tela, temos o nosso Explorador. Logo no primeiro símbolo do VS Code.

Criando um arquivo

É aqui que vamos criar o nosso primeiro arquivo. Para isso, podemos fazer de duas formas. Podemos clicar com o botão direito sobre a área do Explorador e selecionar Novo Arquivo ou podemos também clicar no primeiro símbolo de Novo Arquivo.

Fazendo isso, temos um novo arquivo criado, mas precisamos nomeá-lo. E é muito importante que esse arquivo tenha um nome de index, com um x no final.

O arquivo index.html é o principal arquivo do projeto, o primeiro arquivo que o navegador vai procurar ao tentar abrir uma página web.

Então, é muito importante que utilizemos esse nome nesse momento. Então, vamos chamá-lo de index.html. Depois, basta clicar em Enter. Assim, temos o nosso primeiro arquivo HTML.

Para que os navegadores possam interpretar corretamente a nossa página web, precisamos colocar algumas informações essenciais. Algumas configurações e alguns metadados nesse arquivo index.

Então, precisamos começar fazendo essa base do HTML. Alguns editores, como o nosso VS Code, possuem alguns atalhos que facilitam a criação dessa base. Então, vamos ensinar esse atalho.

E é super interessante que você utilize também com os estudantes. Afinal, são muitas palavras novas para eles colocarem no arquivo nesse momento. Vamos conhecer esse atalho.

Utilizando o atalho do VS Code

Logo na linha 1, vamos escrever html:5, onde 5 é a versão mais recente do HTML. Depois que colocamos o nome, apertamos Enter. E aqui o VS Code cria para nós a base do nosso arquivo index.html.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>

Entendendo o código HTML

Vejam que temos diversas palavras novas, termos em inglês. Então, vamos explicar o que significa cada um desses termos para que vocês conheçam e também consigam explicar para os estudantes, caso eles tenham curiosidade.

A mesma base que acabamos de criar no VS Code está aqui no nosso slide. Vamos utilizar esse material como apoio visual para a nossa explicação.

Logo no topo do nosso arquivo, temos o primeiro elemento, que vamos chamar de tag. Essa primeira tag indica a versão do HTML que estamos utilizando, no caso, a versão 5. Como não indicamos essa versão, por padrão, ele vai pegar a mais recente, que é a 5. Esse primeiro elemento é o doctype.

Logo abaixo, em seguida, temos a segunda tag, o segundo elemento. É ele que vai abrir o nosso arquivo HTML. Logo abaixo, em seguida, temos a próxima tag, que é a abertura do nosso documento HTML. E observem que o nosso documento precisa ser aberto e também fechado. Por isso que temos uma tag de abertura e de fechamento.

Essa tag é o html. E aqui precisamos colocar um atributo muito importante, que é o idioma. Esse atributo é o lang. Aqui, por padrão, temos a língua inglesa, representada ali pelo en. Depois, vamos modificar para pt-br, português do Brasil.

Lá embaixo, fechamos essa tag. E essa estrutura vai se repetir bastante para você. Então, preste atenção. Para fechar qualquer uma dessas tags de conteúdo, usamos a barra do teclado (/) e depois o mesmo nome da tag, que nesse caso é html.

Logo em seguida, temos a tag head. Aqui, esse nome entre o sinal de menor e maior que. É aqui na tag head onde vamos inserir todas as informações necessárias para que os navegadores interpretem corretamente a nossa página. É aqui que colocamos as configurações e também os metadados.

O conteúdo que está dentro do head não é visível para as pessoas usuárias diretamente, e sim para os navegadores.

O nosso conteúdo visível vai vir na nossa próxima tag, que é a body, que, traduzindo de forma literal, seria o corpo da nossa página. É aqui onde vamos inserir todos os textos, títulos e imagens que queremos para a nossa página.

É importante que você e os estudantes compreendam a estrutura e o que temos dentro da tag head. Mas não é tão essencial assim, pois o conteúdo que de fato vamos inserir vem aqui na tag body. É onde vamos inserir todos os textos e também as imagens.

Portanto, não se preocupe tanto nesse momento em decorar os códigos. É mais importante que você entenda a estrutura dessa base HTML.

Vamos para o VS Code nesse momento para fazermos a única alteração que vamos fazer por enquanto aqui dentro da tag head, que é a respeito do title, o título da nossa página.

O title não aparece no corpo da página, mas geralmente vem ali em cima na guia dos nossos navegadores. Então, vamos substituir Document pelo nome do nosso projeto, Hello World. E vamos fazer uma segunda modificação importante. Vamos tirar o en, que representa a língua inglesa, e vamos trocar para pt-br, indicando que vamos utilizar aqui o português do Brasil.

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Hello World</title>
</head>
<body>

</body>
</html>

Lembre-se sempre de salvar o seu progresso no código, para caso você tenha algum problema, você não perca todos os seus arquivos.

E essa é uma dica para você e também para as aulas com seus estudantes. No VS Code você pode utilizar o atalho Ctrl+S, e assim você salva todo o seu projeto e não corre risco de perder alterações.

Mas que tal observarmos o progresso que já tivemos aqui na nossa página? Será que tem algum conteúdo visível? Vamos abrir a pasta do projeto, e preste atenção nesses passos, pois você pode fazer isso sempre que quiser ver como está ficando a sua página web.

Conferindo as modificações na página

Dentro da pasta do projeto temos o arquivo index.html. Precisamos abrir esse arquivo com algum navegador disponível, no nosso caso é o Google Chrome. Clicamos em abrir: temos aqui uma página em branco. Mas observem lá em cima, em nossa guia, que temos o título que acabamos de atribuir lá no nosso arquivo do VS Code.

Agora que entendemos melhor a base do arquivo HTML, queremos trazer uma sugestão para você. Principalmente se esse for o primeiro contato dos estudantes com linguagens escritas, utilize os atalhos como HTML5, pois pode facilitar a mediação da sua aula.

Ao invés dos estudantes passarem bastante tempo escrevendo diversos termos novos, já que não queremos que eles decorem nesse momento, você pode dedicar esse tempo para explicar a estrutura base do HTML.

Subindo o arquivo no GitHub

E por fim, uma dica final: faça o upload desses arquivos lá no repositório do GitHub que já criamos. Então vamos até o repositório para fazer o upload do nosso único arquivo até então, que é o index.html.

Estamos em nossa página inicial do repositório. Para fazermos o upload desses arquivos, vamos vir aqui em Add File, no botão do centro da tela, e depois selecionar Upload Files.

Lembre-se que você pode também traduzir a sua página, caso queira, para o português.

Aqui temos duas opções. A primeira é arrastar todos os arquivos do projeto aqui para dentro. E a segunda é clicar na opção Choose Your Files e escolher os arquivos do nosso computador. É a opção que vamos seguir. Então clicamos na nossa pasta, selecionamos todos os arquivos, ou seja, index.html, e clicamos em Abrir.

Gravem esse nome, commit. Precisamos criar uma nova versão do nosso projeto e descrever de forma bem breve do que se trata. Aqui vamos colocar Criando a Base do HTML.

Assim conseguimos identificar essa versão posteriormente. Poderíamos também inserir algumas informações a mais, só que são opcionais e nesse momento não há necessidade. Para fazermos o commit do projeto, clicamos no botão do canto esquerdo inferior, Commit changes. E assim temos a primeira versão do nosso projeto no GitHub.

Criamos a estrutura base do HTML, e no próximo vídeo vamos começar a inserir o conteúdo textual da nossa página. Então, até a próxima!
